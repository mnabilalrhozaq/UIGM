üß™ Langkah-Langkah Debug

Step 1: Cek Debug Pages

Akses halaman debug untuk memeriksa status sistem:

  • Check Database - Periksa data di database
  • Check Session - Periksa session user
  • Test Save - Test penyimpanan data

Step 2: Login dan Cek Console

  1. Buka browser dalam mode incognito/private
  2. Login: http://localhost:8080/auth/login
  3. Username: admin_unit, Password: password123
  4. Buka Developer Tools (F12) dan pergi ke tab Console
  5. Akses dashboard: Dashboard

Step 3: Periksa Debug Info

Di dashboard, periksa:

  • DEBUG INFO box - Harus menunjukkan "Can Edit: TRUE"
  • Browser Console - Cari pesan debug JavaScript
  • Pastikan tidak ada error merah di console

Step 4: Test Form Save dengan Monitoring

  1. Buka kategori WS (Waste)
  2. Isi form dengan data test:
    • Tanggal Input: Hari ini
    • Gedung: Gedung A
    • Jumlah: 100
    • Satuan: kg
    • Deskripsi: Test data untuk debugging sistem UIGM
  3. Buka tab Network di Developer Tools
  4. Klik tombol "Simpan"
  5. Perhatikan request yang muncul di tab Network

Step 5: Analisis Response

Di tab Network, cari request ke /admin-unit/simpan-kategori:

  • Status Code: Harus 200
  • Response: Periksa isi response JSON
  • Request Payload: Pastikan data terkirim dengan benar

‚ö†Ô∏è Kemungkinan Penyebab Masalah

1. JavaScript Error

  • Cek console untuk error JavaScript
  • Pastikan jQuery loaded dengan benar
  • Periksa apakah event handler terpasang

2. AJAX Request Gagal

  • Cek tab Network untuk melihat request
  • Periksa status code dan response
  • Pastikan URL endpoint benar

3. Server-side Error

  • Cek log file di writable/logs/
  • Periksa error PHP atau database
  • Pastikan model dan controller berfungsi

4. Database Issue

  • Pastikan tabel review_kategori ada
  • Cek foreign key constraints
  • Periksa permission database
